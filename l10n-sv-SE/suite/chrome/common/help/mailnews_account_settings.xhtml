  2. Öppna menyn Redigera och välj Kontoinställningar för E-post & Grupper.
  3. Markera namnet på det konto vars inställningar som du vill visa eller ändra.

  1. Öppna menyn Redigera och välj Kontoinställningar för E-post & Grupper. Du kommer att se dialogrutan Kontoinställningar för E-post & Grupper.
  2. Markera namnet på kontot för att visa panelen Kontoinställningar.
  • Kontots namn: Namnet på kontot.
  • For any type of account but Blogs & News Feeds:
    • Identitet: Lagrar ditt namn, e-postadress, svarsadress (endast om den avviker från din e-postadress) och organisation (valfritt).
    • Signature text: If you want to attach a signature to all outgoing messages, type its text into this box. Check Use HTML to enable HTML code, e.g., <b>bold</b> (optional).
    • Attach the signature from a file instead: Lets you choose to attach the signature from a file (in text, HTML, or image format) rather than entering its text. Checking this option overrides any text entered into the signature box. Click Choose to locate the signature file (optional).

      More signature options are located in Composition & Addressing.

    • Bifoga mitt vCard till meddelanden: Låter dig välja om ditt vCard ska bifogas till dina utgående meddelanden. Klicka på Redigera kort för att redigera uppgifterna i visitkortet (valfritt).
  • For Blogs & News Feeds accounts:
    • Check for new articles at startup: Select this checkbox if you want to check this account automatically for new blogs & news messages whenever you start Mail & Newsgroups.
    • Check for new articles every [__] minutes: Select this checkbox if you want to specify the number of minutes between feed checks. You can also check for new blogs & news messages at any time by clicking Get Msgs in the Mail window.
    • By default, show the article summary instead of loading the web page: Select this checkbox if you want &brandShortName; to display a brief summary of the article (bundled inside the feed) instead of loading the full web page. Showing the article summary is slightly faster than the full web page, but you may miss part of the article content.
    • Empty Trash on Exit: Empties the Trash folder whenever you quit Mail & Newsgroups.
    • Manage Subscriptions...: Shows the Feed Subscriptions dialog, that allows you to add, edit and remove feeds to this blogs & news feeds account.

Kontoinställningar för E-post & Grupper - Serverinställningar

&brandShortName; E-post & Grupper fungerar med två typer av e-postservrar: IMAP och POP. Om du inte är säker på vilken servertyp som din Internetleverantör stöder, fråga din leverantör. Om din Internetleverantör stöder bägge, kan följande beskrivningar kanske hjälpa dig att bestämma vilken du bör använda.

[Återvänd till början av avsnittet]

Om IMAP (Internet Message Access Protocol)

Fördelar: Dina meddelanden och de ändringar du gör i dem förblir på din server, vilket sparar lokalt diskutrymme. Du har också alltid tillgång till en uppdaterad e-brevlåda, och du kan hämta din e-post från ett flertal platser. Prestanda över modem är också snabbare, eftersom du initialt endast hämtar meddelandehuvuden.

Nackdelar: Inte alla Internetleverantörer stöder IMAP.

[Återvänd till början av avsnittet]

Om POP (Internet Post Office Protocol)

Fördelar: Dina meddelanden hämtas till din lokala dator på en gång, men du kan också ange om du vill behålla kopior av meddelandena på servern och ta bort meddelandena på servern när de tas bort lokalt. De flesta Internetleverantörer stöder för närvarande POP.

Nackdelar: Om du använder mer än en dator, kan meddelandena hamna på den ena eller den andra, men inte på båda. POP fungerar inte lika bra som IMAP över en långsam uppkoppling. Du kan inte få tillgång till alla e-postmappar från flera olika platser.

Notera att nyare POP-servrar har funktioner som, i likhet med IMAP, låter dig hämta endast huvudena, i stället för hela meddelandet. Om du uthyttjar dessa funktioner kan prestandan med POP vara nästan lika snabb som med IMAP.

IMAP-serverinställningar

Om du inte redan ser IMAP-serverinställningarna, börja i e-postklientfönstret:

  1. Öppna menyn Redigera och välj Kontoinställningar för E-post & Grupper. Du kommer att se dialogrutan Kontoinställningar för E-post & Grupper.
  2. Markera kontonamnet och klicka på kategorin Serverinställningar. (Om du valde en IMAP-server när du lade upp det här kontot, kommer du att se dina IMAP-serverinställningar.)
  • Servertyp: Servertypen (Server för e-post (IMAP)) som du angav när du skapade det här kontot. För att ändra den till kontot hörande servertypen, måste du ta bort kontot och sedan skapa det på nytt.
  • Server: Servernamnet som du angav när du skapade det här kontot. Om du har problem med att hämta e-post från det här kontot, kontrollera med din leverantör eller systemadministratör att servernamnet som du angav är korrekt.
  • Användarnamn: Det användarnamn som du angav när du skapade det här kontot.
  • Port: Såvida du inte fått andra instruktioner av din leverantör eller systemadministratör, låt den här inställningen vara oförändrad.
  • Connection security: Choose one of the available options to establish a secure connection to your incoming IMAP server. You can choose one of these:
    • None: &brandShortName; will use a plain connection, without encryption at all. You should choose this only if your incoming server doesn't support any type of security.
    • STARTTLS: Require an encrypted connection, use the STARTTLS method. This mechanism will usually run on the standard IMAP port 143.
    • SSL/TLS: Require an encrypted connection, use the IMAP-over-SSL method. The default port for this is 993.
  • Använd säker autentisering: Välj den här inställningen om du vill använda säkra funktioner för att logga in, såsom CRAM-MD5. Om du är osäker huruvida din tjänst stöder detta, kontakta din leverantör eller systemadministratör.
  • Kontrollera om det kommit nya meddelanden vid start: Välj den här inställningen om du vill att E-post & Grupper automatiskt ska kontrollera detta konto om det kommit nya meddelanden, när du startar E-post & Grupper.
  • Kontrollera var [__] minut om det kommit nya meddelanden: Välj den här inställningen för att automatiskt kontrollera om det kommit nya meddelanden, och ange sedan antalet minuter mellan kontroller av ny e-post. Om du inte markerar den här inställningen, kan du söka efter nya meddelanden när som helst genom att klicka på Hämta i e-postklientfönstret.
  • När jag tar bort ett meddelande: Välj den åtgärd du vill ha för borttagna meddelanden. Flytta det till papperskorgen rekommenderas såvida du inte fått instruktioner att använda en annan inställning av din systemadministratör eller leverantör. Meddelanden märkta som borttagna tas bort först när du komprimerar mappar.
  • Rensa (töm) inkorgen vid Avsluta: Avlägsnar borttagna meddelanden från inkorgen när du avslutar E-post & Grupper. Välj detta om du valde att märka meddelanden som borttagna.
  • Töm papperskorgen vid Avsluta: Tömmer papperskorgen när du avslutar E-post & Grupper.
  • Avancerat: Låter dig välja en annan utgående server (SMTP) för utgående meddelanden från detta konto. Du kan även nå Avancerade IMAP-serverinställningar via denna knapp.
  • Lokal sökväg: Den plats på hårddisken där e-post för det här kontot lagras.

Kontoinställningar för E-post & Grupper - Kopior och mappar

Det här avsnittet beskriver inställningarna för att skicka automatiska kopior och för att lagra kopior av utgående meddelanden, utkast till meddelanden och mallar för meddelanden.

Som förval lagrar &brandShortName; E-post & Grupper kopior av dina utgående meddelanden i mappen Skickat för det aktuella kontot. &brandShortName; E-post & Grupper lagrar också utkast till meddelanden i mappen Utkast och meddelandemallar i mappen Mallar för det aktuella kontot.

Om du inte redan tittar på inställningarna för Kopior och mappar, börja i e-postklientfönstret:

  1. Öppna menyn Redigera och välj Kontoinställningar för E-post & Grupper. Du kommer att se dialogrutan Kontoinställningar för E-post & Grupper.
  2. Markera kontot och klicka på Kopior och mappar. Du kommer att se panelen Kopior och mappar.
  • Lägg en kopia i: Välj var kopior ska lagras av dina utgående e-post- och gruppmeddelanden. Om du inte vill använda den förinställda mappen Skickat för det aktuella kontot, klicka på Annan mapp, och välj sedan ett konto och sedan mappen att lagra kopior i.
  • Bcc till dessa adresser: Välj om du alltid vill skicka en dold kopia av meddelandet (bcc) till en annan adressat, och skriv in adressen. Om du alltid vill skicka en dold kopia till dig själv, lägger du till din egen adress till denna lista. Separera adresserna med komma (,).
  • Spara utkast i: Välj var meddelandeutkast ska lagras. Om du inte vill använda den förinställda mappen Utkast för det aktuella kontot, klicka på Annan mapp, och välj sedan ett annat konto och mapp att lagra utkast i.
  • Spara mallar i: Välj var meddelandemallar ska lagras. Om du inte vill använda den förinställda mappen Mallar för det aktuella kontot, klicka på Annan mapp, och välj sedan ett annat konto och mapp att lagra mallar i.
  • Visa varskoendetext när meddelanden sparas: Välj detta alternativ om du vill att E-post & Grupper ska visa en bekräftelsedialogruta när du sparar ett meddelandeutkast eller en mall. Om markerat, kommer en dialogruta att visas när du sparar ett utkast eller en mall för att påminna dig om var &brandShortName; E-post & Grupper sparar utkastet eller mallen.

Inställningar för Diskutrymme (POP)

Meddelanden i POP-konton hämtas alltid i sin helhet till din lokala dator, såvida du inte har aktiverat inställningen Hämta endast brevhuvudet. Det här avsnittet beskriver hur du kan spara utrymme för i POP-konto. Om ditt konto har inställningen Hämta endast brevhuvudet aktiverad, kommer dessa inställningar för Diskutrymme att ignoreras. Om du inte redan tittar på diskutrymmesinställningar för ett POP-konto, följ dessa steg:

Börja i e-postklientfönstret.

  1. Öppna menyn Redigera och välj Kontoinställningar för E-post & Grupper. Du kommer att se dialogrutan Kontoinställningar för E-post & Grupper.
  2. Klicka på kategorin Diskutrymme för ett POP-konto.
  • Meddelanden större än [__] kB: Markera detta alternativ för att bevara diskutrymme genom att hindra stora meddelanden från att hämtas. Ange maximal storlek för hämtade meddelanden.
  • The retention settings can be used to free up space by deleting old messages. Note that these settings apply to both local copies and their originals on the server.

    Note: If your POP account is set up to use a Global Inbox, the retention period settings of the target Inbox apply.

Disk Space Settings (Blogs)

Messages from blogs & news feeds accounts are only stored in your local machine in their short form, ie. the article summary, regardless of whether your settings are to show the full articles by default. Still, there are options to control how much disk space is used by your blogs & news feeds account. If you are not already viewing the Disk Space preferences for a blogs & news feeds account, follow these steps:

Begin from the Mail window.

  1. Open the Edit menu, choose Mail & Newsgroups Account Settings. You see the Mail & Newsgroups Account Settings dialog box.
  2. Click the Disk Space category for a blogs & news feeds account.

There, you can specify which messages should be deleted to recover disk space:

  • Don't delete any messages: Select this option to keep all messages forever. Keep in mind that, if you are subscribed to very high-traffic blogs, this will increase the occupied disk space steadily and could eventually fill up your hard disk.
  • Delete all but the most recent [____] messages: Select this option to keep in each feed only a maximum number of messages. Enter the maximum number of messages (being 1,000 by default).
  • Delete messages more than [__] days old: Select this option to keep in each feed only messages that are not older than the number of days you enter here (being 30 days by default).
  • Always keep flagged messages: Check this option to save (not delete) flagged messages, regardless of its age.

Common Retention Policy Settings

&brandShortName; can automatically delete old messages for you. You can configure this process with the options listed below To recover disk space, old messages can be permanently deleted:

  • Don't delete any messages: Keep all messages. Never delete messages automatically based on their age.
  • Delete all but the most recent [__] messages: Enter the number of messages to keep. With this setting only messages older than these messages are deleted.
  • Delete messages more than [__] days old : Keep all messages that arrived within the given number of days.

With the following settings you can further constrain the three options to delete messages automatically. This is especially useful in combination with the option to keep all messages.

  • Always keep flagged messages: Use this option to deny &brandShortName; to delete any messages you have flagged.
  • Remove bodies from message more than [__] days old: Select this option to retain all headers but to delete message bodies that are older than the number of days you specify here (news accounts only). Any option to delete the entire message based on age still applies.

This policy can be overridden for an individual folder in the Folder Properties, Retention Policy tab.

Note: If message synchronization is enabled (for IMAP), or messages are left on the server (for POP accounts), the settings apply to both local copies and their originals on the server.

Kontoinställningar för E-post & Grupper - Säkerhet

Det här avsnittet beskriver hur du konfigurerar kontoinställningar för E-post & Grupper avseende e-postmeddelandeäkerhet. Innan du kan göra detta måste du emellertid införskaffa ett eller flera e-postcertifikat. För detljer, se Signera och kryptera meddelanden.

Om du inte redan tittar på säkerhetsinställningarna för ditt e-postkonto, börja i e-postklientfönstret:

  1. Öppna menyn Redigera och välj Kontoinställningar för E-post & Grupper.
  2. Klicka på Säkerhet under namnet på e-postkontot vars säkerhetsinställningar du vill konfigurera.
I detta avsnitt:
  • Om certifikat
  • Elektronisk signering
  • Kryptering

Om certifikat

Huvudsyftet med säkerhetspanelen i Kontoinställningar för E-post & Grupper är att välja två certifikat:

  • Det e-postcertifikat som du vill använda för att signera e-postmeddelanden som du skickar till andra personer.
  • Det e-postcertifikat som du vill att andra personer ska använda när de krypterar meddelanden de skickar till dig.

Beroende på policyn hos certifikatutfärdaren (CA) som utfärdar ditt (dina) certifikat, kan du använda ett certifikat för båda syftena eller använda två olika certifikat. Även om du bara använder ett, måste du ange det två gånger, en gång för elektronisk signering och en gång för kryptering.

De certifikat du väljer här infogas med varje signerat meddelande du skickar. Dessa certifikat tillåter dina mottagare att verifiera din elektroniska signatur, och att kryptera meddelanden som de skickar till dig.

Elektronisk signering

Du använder sektionen Elektronisk signering i säkerhetspanelen för att ange hur du vill signera dina e-postmeddelanden:

  • Använd detta certifikat för att signera meddelanden du skickar: Om fältet är tomt eller visar fel certifikat, klicka på Välj för att välja bland de certifikat du har arkiverade.
  • Signera meddelanden: Markera den här rutan ifall du elektroniskt vill signera alla meddelanden du skickar. (Ett eget certifikat måste anges nedan innan du kan markera den här rutan.)

Oavsett om rutan Signera meddelanden är markerad här, kan du ändra dig innan du skickar ett enskilt meddelande.

För att ändra inställningen av elektronisk signatur för ett meddelande du skriver i skrivfönstret, klicka på pilen nedanför säkerhetsikonen nästan högst upp i fönstret, och markera eller avmarkera Signera elektroniskt. För detaljer, se Signera och kryptera ett nytt meddelande.

Kryptering

Du använder sektionen Kryptering i säkerhetspanelen för att ange hur du rutinmässigt vill använda kryptering när du skickar dina meddelanden:

  • Använd detta certifikat för att kryptera och dekryptera meddelanden till dig: Om fältet är tomt eller visar fel certifikat, klicka på Välj för att välja bland de certifikat du har arkiverade.
  • Aldrig: Markera detta alternativ om du aldrig eller endast tillfälligtvis vill använda kryptering.
  • Erfordras: Markera detta alternativ om du alltid vill använda kryptering. Om du inte har alla nödvändiga certifikat, kommer meddelandet inte att skickas såvida du inte uttryckligen stänger av kryptering enbart för det meddelandet.

Oavsett vilket krypteringsalternativ du väljer, kan du ändra dig innan du skickar ett enskilt meddelande.

För att ändra krypteringsinställningen för ett meddelande du skriver i skrivfönstret, klicka på pilen nedanför säkerhetsikonen nästan högst upp i fönstret, och välj den krypteringsinställning som du vill använda. För detaljer, se Signera och kryptera ett nytt meddelande.

Kontoinställningar för E-post & Grupper - Server för utgående (SMTP)

Servern för utgående sänder din utgående e-post till de avsedda mottagarna.

Om du inte redan tittar på inställningarna för Server för utgående (SMTP), börja i e-postklientfönstret:

  1. Klicka på något e-postklientfönster.
  2. Från menyn Redigera, välj Kontoinställningar för E-post & Grupper.
  3. Markera Server för utgående (SMTP), och antingen redigera en befintlig server eller lägg till en ny. Om du inte är säker på vilket alternativ du ska välja, hör med din leverantör eller systemadministratör)
    Du kan välja bland de servrar, vilka nås via rullgardinen för Utgående server i identitetsinställningarna.
  • Beskrivning: En kort fritextbeskrivning av denna serverkonfiguration. Den kommer att visas som den första delen i serverlistan.
  • Server: SMTP-servern som levererar din utgående e-post. För att använda en annan SMTP-server, ändra i detta fält.
  • Port: Den port på SMTP-servern som kommer att användas vid anslutningen. Förvalt visas här standardporten för den angivna krypteringen. Ändra numret om e-postservern lyssnar efter anslutningar till en annan port.
  • Använd namn och lösenord: Om din SMTP-server kräver autentisering för att skicka e-post, markera detta alternativ och ange ditt användarnamn. Första gången du skickar e-post kommer ditt lösenord att efterfrågas. Du kan då instruera &brandShortName; att spara ditt lösenord för framtida sessioner.
  • Använd säker autentisering: Välj denna inställning om du vill använda säkra funktioner för att logga in, såsom CRAM-MD5. Om du är osäker huruvida din tjänst stöder detta, kontakta din leverantör eller systemadministratör.
  • Använd säker anslutning: Det finns två sätt att etablera en säker anslutning till din server för utgående e-post. Välj den som din server stöder. (Om servern inte konfigurerats för det alternativet, kommer du att se ett felmeddelande när du försöker skicka e-post.)
    • STARTTLS, om möjligt: &brandShortName; kommer att försöka använda en krypterad anslutning med hjälp av STARTTLS-metoden. Om servern inte stöder detta, kommer en okrypterad anslutning att användas.
    • STARTTLS: Kräver en krypterad anslutning med hjälp av STARTTLS-metoden. Denna funktion använder vanligen standardporten 25 för SMTP.
    • SMTP-över-SSL: Kräver en krypterad anslutning med hjälp av metoden SMTP-över-SSL (även kallad SMTPS). Standardporten för detta är 465.
